QCoManager Class Reference

Detailed Description

This class provides an interface to the CANopen FD Master library. An instance of QCoManager connects an physical CAN interface to a logical CANopen network. Within this logical CANopen network up to 126 instances of QCoDevice can be handled.

Network management - NMT

In order to control the NMT state of CANopen NMT slave devices the following methods are provided:

NMT transition

Function

Reset device resetDevice()
Switch device to Pre-Operational state switchDevicePreOperational()
Switch device to Operational state switchDeviceOperational()
Switch device to Stopped state switchDeviceStopped()

The current CANopen device state can be evaluated calling deviceState(). This requires a heartbeat cycle time > 0 configured on the CANopen device.

Service Data Object - SDO

Process Data Object - PDO

A Process Data Object (PDO) is added to the CANopen manager manager by addProcessDataObject(). A PDO is defined by the communication direction, identifier, data length code and transmission type. Access to the PDO payload is provided by processData() and setProcessData().

◆ readObject()

Q_INVOKABLE qint32 QCoManager::readObject ( const quint8  ubNodeIdV,
QCoObject *  pclObjectV,
const quint8  ubSdoConfigurationV = 0,
const quint16  uwTimeoutV = 0 
)
Parameters
[in]ubNodeIdV- Node-ID value of CANopen device
[in,out]pclObjectV- Pointer to CANopen object
[in]ubSdoConfigurationV- SDO transfer configuration
[in]uwTimeoutV- SDO response time-out value in milliseconds
Returns
0 on success, other values denote an error

This method reads a CANopen object defined by pclObjectV from a device with a node-ID value of ubNodeIdV. This is a blocking function: it either returns after the SDO transfer has finished or after the SDO response time-out has elapsed. The response time-out value can be changed via the parameter uwTimeoutV. A SDO response time-out value of 0 will use the latest value which has been passed to the CANopen Master Client SDO module. The default value of the CANopen Master Client SDO module is 50 milliseconds.

Depending on the size of the object the SDO service is selected. For an object size of up to 4 bytes the SDO Expedited service is selected. For an object size of more than 4 bytes the SDO Segmented service is selected. In order to use the SDO Block service pass the value eCOM_SDO_CONFIG_BLOCK to the parameter ubSdoConfigurationV.

The method returns 0 after the SDO transfer has finished without an error. A positive return code is interpreted as SDO abort code, using the values of CiA 301 specification (4.2.0). As such, the return value upon a SDO response time-out is SDO_ERR_TIMEOUT. A negative value denotes an error from the CANopen Master library, the return codes are defined by the enumeration ComErr_e.

This method writes a CANopen object defined by pclObjectV to a device with a node-ID value of ubNodeIdV. This is a blocking function: it either returns after the SDO transfer has finished or after the SDO response time-out has elapsed. The response time-out value can be changed via the parameter uwTimeoutV. A SDO response time-out value of 0 will use the latest value which has been passed to the CANopen Master Client SDO module. The default value of the CANopen Master Client SDO module is 50 milliseconds.

Depending on the size of the object the SDO service is selected. For an object size of up to 4 bytes the SDO Expedited service is selected. For an object size of more than 4 bytes the SDO Segmented service is selected. In order to use the SDO Block service pass the value eCOM_SDO_CONFIG_BLOCK to the parameter ubSdoConfigurationV.

The Expedited SDO service of this method uses the command specifier 22h (write request, unspecified length) as default. In order to specify the number of bytes inside the payload for an Expedited transfer pass the value eCOM_SDO_CONFIG_CMD_LEN to the parameter ubSdoConfigurationV.

The method returns 0 after the SDO transfer has finished without an error. A positive return code is interpreted as SDO abort code, using the values of CiA 301 specification (4.2.0). As such, the return value upon a SDO response time-out is SDO_ERR_TIMEOUT. A negative value denotes an error from the CANopen Master library, the return codes are defined by the enumeration ComErr_e.
